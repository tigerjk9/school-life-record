# 생활기록부 점검 웹서비스

NEIS에서 내보낸 학생부 XLS 파일을 업로드하면 Google Gemini AI가 세특·창체·봉사·행특의 **기재요령 위반 여부를 자동 점검**하는 로컬 웹 도구입니다.

> **원본 프로그램 제작**: 대전복수고등학교 박영준 교사
> 원본 Windows 실행 파일(`생기부_DB_생성기(v1.2).exe`, `생기부_AI_Inspector(v1.2).exe`)을 Python 웹 서비스로 재구현한 버전입니다.

---

## 원본 exe 대비 개선 사항

원본 실행기 2개를 분석하고 다음을 반영·보강했습니다.

| 항목 | 원본 exe | 웹 서비스 |
|------|---------|----------|
| 분석 대상 영역 | 세특/창체/봉사/행특 (4개) | 세특/창체/봉사/행특 (4개) ✅ 동일 |
| 실행 환경 | Windows 전용 exe | 로컬 웹(브라우저) ✅ OS 무관 |
| 진행 표시 | 진행률 + 남은 시간 예상 | SSE 실시간 + **남은 시간 예상** ✅ 동일 |
| 위반 근거(Before) | 있음 | 있음(`evidence`) ✅ |
| 수정 제안(After) | 있음 | 있음(`suggested_text`) ✅ |
| 상세 모달 비교 | 있음 | 있음 (클릭 → 모달) ✅ |
| 결과 Excel 내보내기 | 반 단위 append/overwrite | **요약·위반·정상·전체 4시트** ✅ 개선 |
| 학생 조회/키워드 검색 | 없음 | 학년·반·이름·본문 키워드 **전문 검색** ⬆ 추가 |
| 검사 이력 관리 | 엑셀 파일로만 | **DB 기반 복수 이력** (드롭다운 조회) ⬆ 추가 |
| 봉사 기관명 점검 | 부분 지원 | **기관명 전용 프롬프트 기준 명시** ⬆ 개선 |
| 프롬프트 편집 | rules.txt + task_prompt.txt 2개 | 웹 UI에서 직접 편집 + **기본값 복원** ⬆ 개선 |
| 다크 모드 | 없음 | 라이트/다크 토글 ⬆ 추가 |
| DB 자동 백업 | 없음 | 구축 전 `record.db.bak.{timestamp}` 자동 백업 ⬆ 추가 |

## 전체 흐름

```
NEIS XLS 5종 다운로드
    → ① 업로드 탭: 파일 업로드 & DB 구축
    → ② 학생 조회 탭: 학년/반/이름/키워드 검색
    → ③ AI 점검 탭: Gemini로 일괄 검사 (남은 시간 실시간 표시)
    → ④ 결과 확인 탭: 위반 내역 Excel 출력
```

## 설치 & 초기 설정 (최초 1회)

### 사전 준비

- **Python 3.11 이상** — `python --version`으로 확인. 없으면 [python.org](https://python.org)에서 설치 (설치 시 **"Add Python to PATH"** 체크 필수)
- **Git** — `git --version`으로 확인. 없으면 [git-scm.com](https://git-scm.com)에서 설치

### 1단계 — 저장소 클론

```bash
git clone https://github.com/tigerjk9/school-life-record.git
cd school-life-record
```

### 2단계 — 가상환경 생성

```bash
python -m venv .venv
```

### 3단계 — 패키지 설치

```bash
.venv\Scripts\activate
pip install -r requirements.txt
```

> FastAPI, Uvicorn, Pandas, google-generativeai 등 필요한 라이브러리가 모두 설치됩니다. 수 분 소요될 수 있습니다.

### 4단계 — 완료 확인

설치가 끝나면 아래 구조가 갖춰진 상태입니다. 이후부터는 **bat 파일만 실행**하면 됩니다.

```
school-life-record/
├── .venv/               ← 가상환경 (git 제외됨)
├── backend/
├── frontend/
├── requirements.txt
└── 생기부점검_실행.bat  ← 이것만 더블클릭
```

## 매일 시작하기

1. 프로젝트 폴더의 **`생기부점검_실행.bat`** 을 더블클릭합니다.
   가상환경(`.venv`)을 자동으로 활성화하고, 백엔드 서버(포트 8000)를 실행한 뒤 브라우저를 엽니다.
2. 브라우저가 자동으로 열리지 않으면 직접 `http://127.0.0.1:8000` 접속합니다.
3. 종료 시 bat 창을 닫으면 서버도 함께 종료됩니다.

> ⚠️ **보안 주의**: 이 서버는 `127.0.0.1`(내 컴퓨터 전용)로만 실행됩니다. 학생 개인정보가 담긴 서비스이므로 외부 네트워크에 절대 노출하지 마세요.

## NEIS에서 파일 받기

NEIS 교육정보시스템에서 **학생부 항목별 조회(출력)** 메뉴를 이용합니다.

1. NEIS 로그인 → **학생부** 메뉴 진입
2. **학생부 항목별 조회** 또는 **학생부 출력** 선택
3. 학년·반·번호 범위를 지정하고 아래 5개 항목을 각각 엑셀로 내려받습니다.

| 순서 | 항목명 | 비고 |
|------|--------|------|
| ① **필수** | 교과학습발달상황 | 학생 기준 정보 — 반드시 먼저 업로드 |
| ② | 세부능력및특기사항 | 세특 AI 점검 대상 |
| ③ | 창의적체험활동상황 | 창체 AI 점검 대상 |
| ④ | 봉사활동상황 | 봉사 AI 점검 대상 (기관명 포함) |
| ⑤ | 행동특성및종합의견 | 행특 AI 점검 대상 |
| ⑥ | 학년반이력 | 선택 — 학년별 반·번호 이력 보완 |

> 파일 형식은 **.xls 또는 .xlsx** 모두 지원합니다. NEIS 기본 내보내기 형식(.xls)을 그대로 사용하면 됩니다.

## 탭별 사용법

### ① 업로드 & DB 구축

1. ① 교과학습발달상황 카드에 파일을 드래그하거나 클릭하여 선택합니다. *(필수)*
2. 나머지 4개 항목도 같은 방법으로 업로드합니다. *(선택 — 점검할 항목만 올려도 됩니다)*
3. **DB 구축** 버튼을 클릭합니다.
4. 구축 완료 메시지가 나타나면 헤더의 **DB 상태** 뱃지가 초록으로 바뀝니다.

> **DB 초기화** 버튼을 클릭하면 모든 학생 및 점검 데이터를 삭제할 수 있습니다. 학년도가 완전히 바뀔 때 사용하세요.

> DB 구축 시 기존 데이터는 `data/record.db.bak.{timestamp}`로 자동 백업 후 덮어씁니다. 동일 학교·동일 학년도 파일을 새로 올릴 때는 5개 파일을 모두 다시 올리는 것을 권장합니다.

### ② 학생 조회

DB가 구축된 뒤 활성화됩니다.

- **학년 / 반 / 이름** 필터로 특정 학생을 검색합니다.
- **내용 검색** 란에 키워드를 입력하면 세특·창체·봉사·행특 본문을 전문 검색합니다.
- 학생 행을 클릭하면 해당 학생의 항목별 기재 내용을 모달로 확인할 수 있습니다.

### Gemini API 키 발급

AI 점검 기능은 Google Gemini API를 사용합니다. 아래 절차로 키를 발급받으세요.

1. [Google AI Studio](https://aistudio.google.com/) 접속
2. 구글 계정으로 로그인
3. 우측 상단의 **Get API key** 클릭
4. **API 키 만들기** 선택 → 키를 생성하고 복사
5. `AIza...` 로 시작하는 키를 안전한 곳에 보관

> ⭐ **유료 API 사용 강력 권장**: 무료 티어보다 유료(Pay-as-you-go) 버전이 성능과 속도 면에서 훨씬 낫습니다. 무료는 분당 15회 요청 제한으로 오류가 잦습니다. API 키 생성 후 **Google Cloud Console → 결제 설정**에서 결제 계정을 연결하면 **Tier 1(분당 2,000회)**로 업그레이드됩니다. 학급(30명) 점검 비용은 대개 수백 원 수준입니다.
> ⚠️ API 키는 외부에 노출되지 않도록 주의하세요. 이 프로그램은 키를 `data/.apikey` 파일에 저장하며, 서버 재시작 후에도 자동으로 복원됩니다. 키를 변경하려면 API 키 입력란에 새 키를 입력하고 **연결**을 클릭하세요.

### ③ AI 점검

1. **Gemini API 키**를 입력하고 **연결**을 클릭합니다. 연결 상태가 초록으로 바뀌면 성공입니다.
2. 모델 목록에서 사용할 모델을 선택합니다. *Pro > Flash > Flash Lite 순으로 성능/가격. 권장: gemini-2.5-flash*
3. **배치 크기**(1~5)를 설정합니다. 기본값 3은 한 번의 API 호출에 학생 3명분을 묶어 보냅니다. 값이 작을수록 꼼꼼하지만 비용이 높고, 클수록 빠르지만 비용이 적습니다. API 오류가 잦으면 1로 낮추세요.
4. 점검할 **영역**(세특·창체·봉사·행특)과 **학년/반**을 선택합니다. 전체 점검은 비워두면 됩니다.
5. **시스템 프롬프트**를 펼쳐 AI 판단 기준을 확인·수정하거나 **기본값 복원** 버튼으로 되돌릴 수 있습니다.

   > ⚠️ **필수**: 프롬프트 안의 학교명을 **반드시 본교 이름으로 변경**하세요. 변경하지 않으면 본교명이 위반으로 오탐됩니다.

6. **검사 시작**을 클릭합니다. 진행 상황은 실시간 진행률 + **남은 시간 예상**과 함께 표시되고, **라이브 결과** 표에 위반 건이 즉시 나타납니다.
7. 검사 중 **검사 취소**를 누르면 현재 배치가 끝난 후 중단됩니다.

> 점검이 완료되면 **결과 확인 탭**에 자동으로 이력이 저장됩니다.

### ④ 결과 확인

1. **검사 이력** 드롭다운에서 확인할 검사 회차를 선택합니다.
2. 상단 요약 배너에서 위반 건수·정상 건수·사용 모델을 확인합니다.
3. **필터**(전체 / 위반만 / 정상만)로 결과를 걸러볼 수 있습니다.
4. 행을 클릭하면 해당 학생의 원문, AI 판단 근거(위반 유형·사유·근거 문장), 그리고 **수정 제안(After)** 을 확인합니다.
5. **Excel 다운로드**를 클릭하면 요약·위반·정상·전체 4개 시트로 구성된 xlsx 파일을 내려받습니다.

## 주의사항 & FAQ

> **사용 전 필수 체크리스트**
> 1. 시스템 프롬프트에서 **학교명을 본교 이름으로 변경**했는지 확인
> 2. 독서명·봉사활동 기관명이 위반으로 잡혀도 **정상 동작** — 하나씩 확인 필요
> 3. API 오류가 잦으면 **배치 크기 1** 또는 **유료 API 전환**

**Q. 처음 쓸 때 학교명을 어디서 바꾸나요?**
③ AI 점검 탭 → **시스템 프롬프트** 펼치기 → 텍스트 안의 학교명(예: "복수고등학교")을 본교 이름으로 수정 → **프롬프트 저장** 클릭. 되돌리려면 **기본값 복원** 사용.

**Q. 독서명이나 봉사활동 학교명이 위반으로 잡힙니다.**
AI 규칙이 강하게 설정되어 있어 독서 제목, 봉사활동 기관명(학교명 포함) 등이 위반으로 표시되는 경우가 많습니다. 이는 정상 동작이므로 무시하세요. 위반 목록을 하나씩 확인하다 보면 실제 수정이 필요한 내용도 발견됩니다.

**Q. NEIS 파일 컬럼 구조가 달라 오류가 납니다.**
NEIS 업데이트로 컬럼명이 변경될 수 있습니다. 업로드 오류 메시지에 미매칭된 컬럼명이 표시됩니다. 개발자에게 해당 컬럼명을 전달해 파서를 업데이트하면 됩니다.

**Q. NEIS 봉사활동 XLS에 "내용" 컬럼이 없는데 점검됩니까?**
네. 봉사는 내용이 없더라도 기관명(organization)만 있으면 점검 대상에 포함되고, AI가 기관명을 기준으로 위반 여부를 판단합니다.

**Q. API 호출 중 오류가 자주 발생합니다.**
무료 티어의 분당 요청 한도(15 RPM) 초과가 원인일 수 있습니다. **배치 크기를 1로 줄이면** 호출 빈도가 낮아져 안정적으로 동작합니다. 근본적인 해결책은 Google Cloud에서 **결제 계정을 연결**하여 유료 티어(2,000 RPM)로 업그레이드하는 것입니다. 재시도는 지수 백오프로 자동 처리됩니다.

**Q. AI 결과가 실제 규정과 다릅니다.**
AI는 오탐(False Positive)을 낼 수 있습니다. 결과는 **참고 용도**로만 사용하고, 교사가 원문과 함께 최종 판단하세요. 시스템 프롬프트를 수정해 판단 기준을 조정할 수 있으며, 편집 후 **기본값 복원** 버튼으로 원본 프롬프트로 되돌릴 수도 있습니다.

**Q. 프로그램을 종료했다가 다시 실행하면 데이터가 사라집니다.**
데이터는 `data/record.db` SQLite 파일에 영구 저장됩니다. 파일이 존재하면 재실행 후에도 DB 상태가 유지됩니다. 데이터를 초기화하려면 해당 파일을 삭제하세요.

**Q. 학년도가 바뀌면 어떻게 합니까?**
새 학년도 파일을 업로드하고 DB 구축을 다시 실행하면 됩니다. 이전 데이터는 `data/record.db.bak.{날짜}`로 자동 백업됩니다.

**Q. 학년 일부만 점검할 수 있나요?**
네. 예를 들어 1학년 전체를 이미 점검했다면, DB를 업데이트한 뒤 AI 점검 탭에서 **학년/영역만 선택**하여 추가 점검할 수 있습니다. 전체를 다시 점검할 필요가 없습니다.
