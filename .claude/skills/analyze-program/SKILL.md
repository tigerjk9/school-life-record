# 생활기록부 프로그램 분석 스킬

## 분석 대상

### 핵심 파일
- `생활기록부 점검 프로그램 사용법.pptx` (또는 `_extracted/manual.pptx`)
- ZIP 내 EXE 2종: `생기부_DB_생성기.exe`, `생기부_AI_Inspector.exe`

### 프로그램 구조 (이미 파악된 내용)
1. **DB 생성기**: NEIS XLS 5종 → `record.db` (SQLite)
2. **AI Inspector**: `record.db` + Gemini API → 점검 결과 Excel

## NEIS XLS 파일 5종 예상 구조

| 파일 종류 | 주요 컬럼 추정 |
|----------|--------------|
| 교과성적 | 학년, 반, 번호, 이름, 과목명, 원점수, 성취도, 석차등급 |
| 세부능력및특기사항(세특) | 학년, 반, 번호, 이름, 과목명, 세특내용 |
| 창의적체험활동(창체) | 학년, 반, 번호, 이름, 영역, 활동내용, 시간 |
| 봉사활동상황 | 학년, 반, 번호, 이름, 기관명, 활동내용, 시간 |
| 행동특성및종합의견 | 학년, 반, 번호, 이름, 담임교사, 내용 |

> 실제 파일로 컬럼 검증 필요. 위는 추정값.

## 분석 수행 방법

### PPTX 슬라이드 텍스트 추출
```bash
unzip -p manual.pptx 'ppt/slides/slide*.xml' | grep -o '<a:t>[^<]*</a:t>' | sed 's/<a:t>//g; s/<\/a:t>//g'
```

### 기능 목록 도출 체크리스트
- [ ] XLS 업로드 순서 (교과성적 먼저 필수)
- [ ] DB 업데이트 방식 (덮어쓰기 vs 누적)
- [ ] 학생 조회 필터 (학년, 반, 번호, 이름, 내용검색)
- [ ] AI 점검 설정 (배치 크기 1~5, 모델 선택)
- [ ] 작업 프롬프트 구조 (대전제, 목표, 작업방향, 출력형식)
- [ ] 결과 필터 (전체/위반만/정상만)
- [ ] 위반 상세 모달 내용

```
# 기능 요구사항

## XLS 스키마 (검증 필요 항목 명시)
```
